ChatGPT 新模型 GPT-5.3 Instant：不再让你"冷静一下"
21.2K 条 X 讨论

OpenAI 发布了一项更新，旨在让 ChatGPT 最常用的模型在回复时感觉更自然、更少人工痕迹。

ChatGPT 用户经常遇到这样的回复："深呼吸一下"、"你并没有问题（You're not broken）"、"有这种感觉是正常的"——这些话本意是表示支持，但往往被当成居高临下的说教，尤其是在中立的对话场景中。

OpenAI 新发布的 GPT-5.3 Instant 模型正是针对这一反馈，减少了公司所称的"尴尬因子（cringe factor）"。公司表示："我们听到了你们的反馈，5.3 Instant 减少了那些让人尴尬的回复。"

GPT-5.2 的问题

旧版 GPT-5.2 Instant 有一个默认习惯——假设用户处于情绪困境中。即使用户只是在问一个关于做饭或编程的普通问题，它的第一句话也会是"首先——你并没有问题（First of all — you're not broken）"。这种自以为是的"心理辅导"式开场白一直让用户火大。

Reddit 和社交媒体上的用户将 GPT-5.2 的回复描述为"把人当婴儿"和"过度小心"。有一位用户总结了大家的心声："在整个人类历史上，从来没有人因为被告知'冷静一下'而真正冷静下来过。"

用户的不满不是关于消除同理心，而是关于语境的适当性——问技术问题的用户不需要心理治疗式的语言。这种挫败感在 2026 年初达到了临界点，大量报告显示用户因为 AI 的语气而直接取消了订阅。

GPT-5.3 Instant 的核心改进

新版本的目标是提供"更准确的回答，在网页搜索时提供更丰富、更有上下文的结果"。具体改进包括：

· 语气改进：更新消除了戏剧化的用语，采用更自然、更聚焦的对话风格。减少了 AI 拒绝回答适当问题或在敏感话题前过度道德说教的情况。

· 大幅减少不必要的拒绝回答：削弱了过度防御或说教的前置铺垫——模型现在会直接聚焦用户问题，不再加无谓的免责声明。

· 更好的网页搜索整合：更新改进了网页搜索数据融入回复的方式，让信息感觉更无缝地融入回答，而不是尴尬地附加上去。

· 减少幻觉：在高风险评估中，GPT-5.3 Instant 使用网页搜索时幻觉率降低了 26.8%，仅依赖内部知识时降低了 19.7%。在用户反馈评估中，使用网页搜索时幻觉减少了 22.5%，不使用网页时减少了 9.6%。

平衡的艺术

OpenAI 面临一个走钢丝的挑战：同理心太少会显得冷漠，同理心太多则显得虚假。公司在保持安全护栏的同时改进了语气，除非有证据表明用户需要情感支持，否则默认假设用户是有能力的成年人。这个系统不再简单地过滤特定短语，而是更好地理解用户意图，选择语境上适当的回复风格。

行业大趋势

这次更新反映了一个更大的趋势：用户越来越重视 AI 工具的能力、清晰度和效率，而不是家长式的情感智能。即日起面向所有 ChatGPT 用户开放。

阿里巴巴 Qwen 技术负责人林俊阳与两名同事一同离职
14.5K 条 X 讨论

阿里巴巴 Qwen AI 项目的技术负责人林俊阳（Junyang Lin）通过 X 宣布离职，只留下一句简短的话："me stepping down. bye my beloved qwen."（我辞职了。再见，我亲爱的 Qwen。）这条公告在 AI 社区引发了巨大反响，获得了数百次转发和数千个点赞。

林俊阳不只是一名幕后研究员，他是 Qwen 项目在社交媒体上的公众形象。他定期分享模型发布信息、基准测试结果，并与全球开发者互动。正是他的活跃存在，让这个中国 AI 项目在国际受众面前有了一个可辨识的人格化形象。

其他离职人员

这次离职并非孤例。团队另一名研究员李凯鑫（Kaixin Li）发帖称："正式告别 @Alibaba_Qwen。感谢有机会与如此杰出的头脑们共事。为我们的影响力感到骄傲。继续向前！" 惠彬元（Binyuan Hui）的个人主页也改成了"前 Qwen 成员技术人员"的状态。此外，据晚点 LatePost 报道，负责后训练（post-training）方向的余博文也在同一天辞职。

并非自愿？

Qwen 贡献者陈昌的一条帖子暗示林俊阳的离开并非自愿，他说："我知道离开不是你的选择。就在昨晚，我们还肩并肩地一起发布 Qwen 3.5 小模型。"

林俊阳的贡献

林俊阳 2019 年加入阿里巴巴，历经晋升成为高级算法专家（Senior Staff Algorithm Engineer），2023 年起正式担任 Qwen 技术负责人。他此前的重要项目包括 M6、OFA 和 Chinese-CLIP。根据 Google Scholar 的数据，他的论文累计被引用超过 42,000 次，仅 Qwen3 技术报告一篇就获得约 9,000 次引用。

Qwen 的全球影响力

在林俊阳的带领下，Qwen 从 2023 年最初的 70 亿参数版本发展成一个全面的产品组合——涵盖语言模型、视觉-语言变体、语音系统和专业推理模型。整个生态系统在 Hugging Face 上实现了超过 6 亿次下载和超过 17 万个衍生版本，在该指标上超过了 Meta 的 Llama。《财富》杂志在 2025 年"改变世界"榜单中表彰了阿里巴巴对开源 AI 的贡献。

关于中国 AI 结构性制约的公开发言

林俊阳曾公开坦承中国 AI 研究面临的结构性挑战，指出美国的算力基础设施可能比中国高出一到两个数量级。他强调，中国的实验室往往将资源主要用于完成交付任务，而非开拓性的研究方向。他同时表示，算法创新是弥补这一差距的补偿策略。

他未来的计划尚未公开，但鉴于他的知名度，他接下来的动向必将引起广泛关注。

GPT-5.4 被泄露：OpenAI Codex 代码提交暴露上下文窗口、视觉能力和发布时间线
X 热传中

2026 年 3 月 3 日 —— OpenAI 并不打算告诉我们关于 GPT-5.4 的事。但代码提交、一张被删除的截图、以及一个短暂可见的 API 端点，已经勾勒出了即将到来的东西的清晰画面。

在过去一周内，三次独立的泄露确认了 GPT-5.4 在 OpenAI 内部已经存在，而且已经成熟到可以在生产代码中被引用。证据指向一个拥有 200 万 token 上下文窗口、全分辨率图像处理和更强代理能力的模型——相比仅在几周前发布的 GPT-5.3-Codex，这是一次重大升级。

以下是我们知道的一切、它意味着什么、以及可以期待什么。

三次泄露：发生了什么

泄露 1：Codex GitHub 提交

一名 OpenAI 工程师向公开的 Codex 仓库提交了一个 pull request，其中包含一个版本检查，引用了"GPT-5.4 or newer"作为新图像处理功能的要求。社区在数小时内发现了它。

这行代码很快被改为"gpt-5.3-codex or newer"，提交历史被 force-push 覆盖——但截图已经在 X 和 Reddit 上广泛传播。

为什么这很重要：这不是一个占位符。代码正在实现一个特定功能——全分辨率图像处理——这需要 GPT-5.4 的能力。工程师写这个版本检查是因为该功能在旧模型上根本无法工作。

泄露 2：模型选择器截图

一名叫 Tibo 的 OpenAI 员工意外发布了 Codex 应用模型选择界面的截图。GPT-5.4 作为可选选项与 GPT-5.3-Codex 并列显示。帖子很快被删除，但截图已经被保存和分享。

泄露 3：Alpha API 端点

X 用户 @nicdunz 报告称"alpha-gpt-5.4"短暂出现在一个公开的 /models API 端点中。虽然这一说法缺乏独立验证，但它与 OpenAI 在发布前将模型部署到 alpha 端点的惯例一致。

GPT-5.4 的样貌（基于证据）

200 万 token 上下文窗口

最重大的升级。具体来说，200 万 token 的上下文窗口意味着：

· 整个大型代码库可以放入单次会话中——不再需要分块

· 完整的书籍、法律文件或研究论文可以不拆分地处理

· 长时间运行的代理会话能在数小时工作中保持连贯性

· 多文件重构时可以拥有每个受影响文件的完整上下文

这是从 GPT-5 已经很大的 40 万窗口起 5 倍的跃升，将为生产级 AI 模型设立新标准。

全分辨率视觉

泄露的代码增加了使用新的 "detail": "original" 参数直接向 Responses API 传递原始图像字节（PNG、JPEG、WebP）的支持。

当前的视觉模型在处理前会压缩图像，这会降低以下场景的质量：架构图和流程图、带有细小文字的 UI 截图、医学影像、卫星和航拍图像、技术图纸。全分辨率处理将使 GPT-5.4 在需要像素级精度的视觉任务上能力显著增强。

增强的代理能力

报告描述了将 GPT-5.4 推向"真正能独立完成多步骤任务的代理"的改进。虽然具体细节有限，但这与 OpenAI 的发展轨迹一致：

· GPT-5.2：擅长单个任务

· GPT-5.3-Codex：为自主编程和终端使用而优化

· GPT-5.4：跨编程、研究和视觉任务的更广泛自主能力

GPT-5 系列时间线：加速的发布节奏

发布节奏正在加速。OpenAI 在 6 个月内发布的 GPT-5 变体比整个 GPT-4 时代一年内的还多。如果泄露的引用准确的话，GPT-5.4 可能在数周内发布。

对 AI 版图的意义

上下文窗口竞赛：如果 GPT-5.4 以 200 万上下文窗口发布，它会迫使竞争对手做出回应。Google 的 Gemini 2.5 Pro 目前拥有最大的 100 万 token 上下文窗口。GPT-5.4 的 200 万将使这一领先翻倍。
